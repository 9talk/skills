启动一个新的规划。

**输入**：用户的请求应当包含变更名称或对想要构建内容的描述。

**步骤**
1. **如果没有提供明确的输入，询问用户想要构建什么**
   使用 **AskUserQuestion Tool**（开放式，无预设选项）询问：
   > "您想要进行什么规划？请描述您想要构建或修复的内容。"
   根据描述，提炼出简洁的中文规划主题（例如：检索列表页）。如果输入少于 50 字，直接使用用户输入作为规划主题；否则要求用户输入规划名称。
2. **创建新规划**
   调用 `bash ~/.claude/skills/pl/scripts/pl-init.sh` 脚本，传入规划名称以创建新的规划文件。
3. **如果没有具体规划输入**
   - 输出规划开始共创的提示，邀请用户一起讨论和细化规划内容
   - 示例提示："规划已创建，让我们开始共创吧！您希望从哪里开始讨论？"
4. **共同规划**
   - 使用 **Plan 模式** 探索相关内容（代码库结构、相关功能模块等）
   - 润色用户的原始输入，提炼出更清晰、更具体的规划描述
   - 返回润色后的规划内容供用户确认

## 要求
- 本技能仅支持创建新规划，不处理已有规划的读取或修改。
- 新规划必须通过 `bash ~/.claude/skills/pl/scripts/pl-init.sh` 脚本创建。
- 禁止覆盖现有规划文件。
- 现在处于规划设计阶段,可以探索相关内容（代码库结构、相关功能模块等）。
- 严禁编写任何具体的代码。我们只讨论规划。如果我没明确说‘开始编码’，请不要输出具体实现逻辑。
